# Demo assets

This directory holds the scripted asciinema recording embedded in the
project README.

- `demo.py` — pexpect driver that types the demo into a fresh `bash`
  inside `asciinema rec`. PEP 723 standalone script; runs under `uv`.
- `demo.cast` — the recorded session (asciicast v3). Regenerated by
  running the driver.

## One-time setup

Both prerequisites are user-level CLIs:

```sh
cargo install asciinema       # or: brew install asciinema
curl -LsSf https://astral.sh/uv/install.sh | sh   # if you don't have uv
```

`uv` handles `pexpect` itself — the dependency is declared inline at the
top of `demo.py`.

## Regenerate the cast

```sh
cargo build --release -p mdvs
./target/release/mdvs build example_kb
uv run assets/demo.py
```

The script overwrites `assets/demo.cast` in place.

## Play it back locally

```sh
asciinema play assets/demo.cast
```

## Render to GIF for the README

```sh
cargo install --git https://github.com/asciinema/agg
agg assets/demo.cast assets/demo.gif --theme monokai --font-size 16
```
