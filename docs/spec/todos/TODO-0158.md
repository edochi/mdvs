# TODO-0158: --where translator does not handle quoted/special-character field names

## Summary

The schema-aware `--where` translator (`cmd/search.rs` → `index/backend.rs::
translate_where_to_struct`, TODO-0016 wave 2) rewrites bare identifier chains
to `data.<path>` using a regex over `[A-Za-z_][A-Za-z0-9_]*`. It does not
understand SQL **quoted identifiers** (`"lab section"`) or field names
containing spaces, quotes, or other punctuation. Such a clause silently
matches nothing instead of filtering correctly.

## Details

example_kb has fields with exotic names: `lab section` (space), `notes"v2"`
(embedded quotes), `author's_note` (apostrophe). A user filtering on them, e.g.

```
mdvs search q --where '"lab section" = "A"'
```

produces a translation like `"data.lab data.section" = ...` (the inner words
are matched and prefixed independently; the surrounding double quotes are not
recognized as a quoted-identifier delimiter), which references no real column
and returns zero hits — with no error.

### Why it happens

- The identifier regex matches `[A-Za-z_][A-Za-z0-9_]*(?:\.[A-Za-z_]...)*` and
  has no notion of double-quoted identifiers.
- Single-quoted **string literals** are protected (skipped), but
  double-quoted **identifiers** are not.

### Possible fixes

- Extend the literal/skip pass to also recognize double-quoted identifiers,
  and when one wraps a frontmatter field, rewrite to ``data."lab section"`` (or
  the LanceDB-accepted nested form) rather than tokenizing the inside.
- Verify what LanceDB/DataFusion accepts for struct-child access when the
  child name contains spaces (likely backtick or double-quote quoting of the
  segment — spike it).
- Add translator unit tests + an integration test on a field named with a
  space.

### Why low priority

Field names with spaces/quotes are rare and discouraged; the common case
(identifier-safe names, including dotted nested paths) works. Workaround today:
none clean for spaced names — rename the frontmatter key. Revisit if a real
vault needs it.

## Files likely touched

- `crates/mdvs/src/index/backend.rs` — `translate_where_to_struct` regex/pass.
- Tests in `backend.rs` (translator units) + `cmd/search.rs` (integration).
