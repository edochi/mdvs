# TODO-0075: Support array containment queries in --where

## Summary

There's no intuitive way to query "array field contains value" via `--where`. Array fields like `concepts: ["[[QUIC]]", "[[WebRTC]]"]` can't be filtered with simple equality — users need to know DataFusion-specific array functions.

## Details

Investigate what DataFusion supports for array queries:
- `array_contains(concepts, '[[QUIC]]')` or `array_has(concepts, '[[QUIC]]')`
- `'[[QUIC]]' = ANY(concepts)`
- `unnest(concepts)` — row expansion

Depending on what works, options include:
1. **Document it** — show users the correct DataFusion syntax for array queries
2. **Promote array children in the VIEW** — similar to how Struct children are promoted, though semantics differ
3. **Helper syntax** — preprocess `--where` to expand a shorthand like `concepts HAS '[[QUIC]]'` into the DataFusion equivalent

First step: test what DataFusion actually supports on `FixedSizeList`/`List` columns and document findings.

## Files

- `src/search.rs` — VIEW creation, query construction
- `src/index/backend.rs` — SQL query building
