# TODO-0093: Verbose text output — process step lines on success

## Summary

Update `format_text` on each command so that verbose mode (`--output text -v`) renders process step lines (with elapsed times) before the result content. Currently verbose only affects the result detail level, not whether process steps are shown.

## Details

### Rules

- **No errors, compact** (`--output text`): result-derived content only (current behavior)
- **No errors, verbose** (`--output text -v`): process step lines followed by result content
- **Errors** (always, regardless of verbose): process info highlighting where the pipeline stopped (current behavior)

### Design questions

- What format for step lines? e.g. `read_config ✓ (2ms)`, `scan ✓ 43 files (15ms)`, `embed_files ✓ 12 files (1.2s)`
- Should skipped steps be shown? e.g. `load_model — skipped`
- How much detail per step? Just name + elapsed, or also a short summary from the step output?

- `src/cmd/search.rs` — same
- `src/cmd/info.rs` — same
- `src/cmd/clean.rs` — same
