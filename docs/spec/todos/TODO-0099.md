# TODO-0099: Redesign auto-update/auto-build pipeline across commands

## Summary

Restructure how commands chain into each other. The principle: **downstream commands can auto-run upstream steps, not the other way around.** The dependency chain is `update → build → search`, so only `build` and `search` should offer auto-inclusion of upstream steps. Remove auto-build from both `init` and `update`.

## Current State

- `init`: auto-builds by default, `--suppress-auto-build` to skip. Also accepts `--model`, `--revision`, `--chunk-size` for the embedded build step.
- `update`: `[update].auto_build = true` by default, `--build` flag to override.
- `build`: no auto-update, assumes schema is current.
- `search`: no auto-update, no auto-build, queries whatever's in `.mdvs/`. Stale index is silent (TODO-0089).

## Proposed Design

### Dependency chain

```
update (schema) → build (index) → search (query)
  upstream                          downstream
```

Each command can optionally run the steps above it in the chain. No command auto-runs steps below it.

### Command changes

#### `init`
- **Remove auto-build entirely.** Init becomes schema-only: scan → infer → write `mdvs.toml` (validation sections only: `[scan]`, `[update]`, `[fields]`).
- **Remove flags:** `--suppress-auto-build`, `--model`, `--revision`, `--chunk-size`. These belong to `build`.
- **Never writes** `[embedding_model]`, `[chunking]`, `[search]` sections. `build` fills them with defaults on first run (this mechanism already exists).
- **No model download** during init. Users who only need validation (`init` + `check`) never touch embeddings.

#### `update`
- **Remove `[update].auto_build`** and the `--build` flag. Update becomes pure inference: scan → infer → merge fields → write `mdvs.toml`. No build step.
- **`[update]` section** keeps only non-auto-build settings (currently empty after this change — may be removed or kept as a placeholder for future settings).

#### `build`
- **Add `auto_update`** setting in a new config location (see Config section below).
- **Add `--no-update` flag** to suppress auto-update for that invocation.
- Default behavior: `build` → auto-update (scan + infer + merge fields) → validate → classify → embed → write index.
- With `--no-update`: validate → classify → embed → write index (current behavior).

#### `search`
- **Add `auto_update` and `auto_build`** settings in `[search]` section (both default `true`).
- **Add `--no-update` and `--no-build` flags** to suppress for that invocation.
- Default behavior: `search` → auto-update → auto-build → query. One command, always fresh.
- With `--no-build`: auto-update → query existing index (fresh schema, stale results).
- With `--no-update --no-build`: just query (current behavior).
- With `--no-update`: auto-build (with current schema) → query.

### Config changes

#### `[update]` section
Remove `auto_build`. Section may become empty — keep it as a placeholder or remove entirely. If kept, it's where future update-specific settings would go.

#### `[build]` section (new)
Currently build config is spread across `[embedding_model]` and `[chunking]`. Add a `[build]` section for build-specific workflow settings:

```toml
[build]
auto_update = true   # run update before building
```

Or alternatively, put `auto_update` directly on the existing sections. The `[build]` section approach is cleaner — separates workflow config from model/chunking config.

#### `[search]` section
Currently has `top_k`. Add:

```toml
[search]
top_k = 10
auto_update = true   # run update before building (when auto_build is true)
auto_build = true    # run build before searching
```

### Flag naming

Use `--no-update` and `--no-build` (short, standard negation pattern). Consistent across `build` and `search`.

### Combination matrix

| Command | `auto_update` | `auto_build` | Pipeline |
|---|---|---|---|
| `init` | — | — | scan → infer → write toml |
| `update` | — | — | scan → infer → merge fields → write toml |
| `build` | true (default) | — | update → validate → classify → embed → write index |
| `build --no-update` | false | — | validate → classify → embed → write index |
| `search` | true (default) | true (default) | update → build → query |
| `search --no-build` | true | false | update → query stale index |
| `search --no-update` | false | true | build → query |
| `search --no-update --no-build` | false | false | query only |

### First-time experience

```bash
mdvs init example_kb          # schema only, instant, no model download
mdvs search "query" example_kb  # auto-update (no changes) → auto-build (download model, embed) → query
```

Model download happens on first `build` or `search`, not `init`.

### Validation-only workflow

```bash
mdvs init example_kb    # write mdvs.toml
mdvs check example_kb   # validate — no model, no .mdvs/ ever created
```

No embedding involvement at all.

## Impact

### Code changes
- `src/cmd/init.rs` — remove build pipeline, remove `--suppress-auto-build`/`--model`/`--revision`/`--chunk-size` flags
- `src/cmd/update.rs` — remove build pipeline, remove `--build` flag
- `src/cmd/build.rs` — add auto-update step, add `--no-update` flag
- `src/cmd/search.rs` — add auto-update + auto-build steps, add `--no-update`/`--no-build` flags
- `src/schema/config.rs` — add `[build]` section with `auto_update`, add `auto_update`/`auto_build` to `[search]`, remove `auto_build` from `[update]`
- `src/main.rs` — update clap definitions

### Config migration
- Existing `[update].auto_build = true` in user toml files becomes a no-op (or warning). The setting moves to `[build].auto_update` and `[search].auto_update`/`[search].auto_build`.
- New toml files written by `init` won't have `[embedding_model]`, `[chunking]`, or `[search]` sections until first `build`.

### Subsumed TODOs
- **TODO-0089** (stale index warning): No longer needed — `search` auto-builds by default, so the index is always fresh. Users who disable auto-build (`--no-build` or config) are explicitly choosing stale results.
