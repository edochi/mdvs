# TODO-0006: Support enum constraints on string fields

## Summary

Add an `enum` constraint to String fields that restricts values to a declared set of allowed strings. Not a new FieldType — enums are strings with a validation constraint.

## Details

**Toml representation:**
```toml
[[fields.field]]
name = "format"
type = "String"
enum = ["multiple-choice", "will-it-compile", "free-form"]
```

**Inference:** Detect low-cardinality string fields automatically. If a string field has few distinct values relative to its file count (heuristic TBD), suggest it as an enum candidate. Could be opt-in via `--infer-enums` or similar.

**Validation (check):** If `enum` is set, check that the value is in the allowed set. New ViolationKind or extend WrongType with detail like "got 'foo', expected one of [...]".

**Storage:** Remains a String column in Arrow/Parquet. No storage change.

**Query:** No change — WHERE clauses on string fields already work.

**Files:** `src/schema/config.rs` (add `enum` to TomlField), `src/cmd/check.rs` (validate enum constraint), `src/discover/infer.rs` (optional enum inference)
