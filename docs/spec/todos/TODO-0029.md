# TODO-0029: User documentation site with mdBook

## Summary

Create a user-facing documentation site using mdBook, deployed to GitHub Pages. Uses `example_kb/` as the running example throughout. The README covers installation and quick start; the docs site covers full usage, configuration reference, and features in action.

## Details

### Why

The README should stay concise (install + quick start + feature overview). Users need a full reference for:
- All 7 commands with flags, examples, and behavior
- `mdvs.toml` configuration reference (all sections, all fields, defaults)
- Frontmatter schema: types, inference, widening, validation rules
- `--where` SQL filtering syntax and examples
- Incremental builds, model management, migration between versions
- Recipes / how-to guides (e.g. Obsidian vault setup, CI integration)

### Structure

```
book/
├── book.toml
└── src/
    ├── SUMMARY.md
    ├── introduction.md          # what mdvs is, the "DB of documents" pitch
    ├── getting-started.md       # install + first run on example_kb
    ├── concepts.md              # schema inference, types, widening, validation
    ├── commands/
    │   ├── init.md
    │   ├── check.md
    │   ├── update.md
    │   ├── build.md
    │   ├── search.md
    │   ├── info.md
    │   └── clean.md
    ├── configuration.md         # full mdvs.toml reference
    ├── search-guide.md          # --where syntax, ranking, examples
    └── recipes/
        ├── obsidian.md
        └── ci.md
```

### Key decisions

- **Location:** `book/` at repo root (not `docs/book/`). `docs/` is implementor-facing specs; `book/` is user-facing documentation. Different audience, different purpose.
- **Running example:** All pages use `example_kb/` (the Prismatiq Lab fixture) for examples and output snippets. This keeps examples consistent and testable.
- **Flat over nested:** `concepts.md` covers types, inference, widening, and validation in one page (they're tightly connected). `search-guide.md` covers `--where` syntax and ranking together. Avoids forcing readers to jump between small pages.
- **Query examples in depth:** `search-guide.md` must include detailed, runnable examples for every query pattern: scalar filters, array containment (`array_has`, `= ANY()`), array length, nested object bracket access, field names with special characters, and combined filters. All examples against `example_kb/` so readers can copy-paste and try them. See `scripts/test_array_queries.rs` for tested query patterns to draw from.

### Steps

1. `cargo install mdbook`
2. Initialize `book/` with `mdbook init`
3. Write content — rewrite from `docs/spec/commands/` for users, not implementors. All examples from `example_kb/`.
4. GitHub Actions workflow to build and deploy to GitHub Pages on push to main
5. Link from README: "Full documentation at ..."
