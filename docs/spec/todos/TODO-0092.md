# TODO-0092: Compact JSON output — result-only when no errors

## Summary

Add `format_json(verbose: bool)` to the `CommandOutput` trait so that compact JSON (`--output json` without `-v`) emits only the `result` key, while verbose or error states emit the full struct (process + result).

## Resolution

Added `format_json(verbose)` to the `CommandOutput` trait with a default implementation that serializes the full struct. Added `format_json_compact` helper in `output.rs` that omits `process` when `result.is_some() && !verbose`. All 7 `*CommandOutput` impls override `format_json` via the helper. Removed `skip_serializing_if` from `result` fields on init/build/update so error cases show `"result": null`.
