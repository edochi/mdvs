# TODO-0092: Compact JSON output — result-only when no errors

## Summary

Add `format_json(verbose: bool)` to the `CommandOutput` trait so that compact JSON (`--output json` without `-v`) emits only the `result` key, while verbose or error states emit the full struct (process + result).

## Details

### Rules

- **No errors, compact** (`--output json`): `{"result": {...}}`
- **No errors, verbose** (`--output json -v`): `{"process": {...}, "result": {...}}`
- **Errors** (always, regardless of verbose): `{"process": {...}, "result": null}`

### Implementation

1. Add `format_json(&self, verbose: bool) -> String` to `CommandOutput` trait in `src/output.rs`
2. Update the default `print()` method to call `format_json(verbose)` instead of raw `serde_json::to_string_pretty`
3. Each `*CommandOutput` implements `format_json`:
   - If `result.is_some() && !verbose` → serialize `{"result": ...}` only
   - Otherwise → serialize full struct (process + result)

- `src/output.rs` — add `format_json` to `CommandOutput` trait, update `print()`
