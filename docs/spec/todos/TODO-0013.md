# TODO-0013: Search verifies model against parquet metadata

## Summary

Search must compare the model/revision in mdvs.toml against what's stored in parquet metadata. Hard error on mismatch — prevents silently comparing embeddings from different models.

## Details

- Before loading the model, read `BuildMetadata` from chunks.parquet
- Compare `config.embedding_model.name` vs `metadata.model` and `config.embedding_model.revision` vs `metadata.revision`
- On mismatch: hard error with a message like "model in config (X) does not match index (Y) — run 'mdvs build --force' to rebuild"
- If no metadata exists (old index): warn but proceed (backwards compat for existing indexes built before this feature)
