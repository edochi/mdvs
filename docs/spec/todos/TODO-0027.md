# TODO-0027: Prefix internal parquet columns to avoid frontmatter collisions

## Summary

Rename internal columns in `files.parquet` and `chunks.parquet` with a configurable prefix (default `_`) so that frontmatter fields can be promoted to top-level columns without name collisions.

## Details

### Problem

Frontmatter fields are stored in a `data` Struct column. To expose them as top-level columns in search (see TODO-0028), we need to avoid collisions with internal column names like `file_id`, `filename`, `content_hash`, `built_at`.

### Solution

1. **Add `internal_prefix` config** — new field, not in any existing section. Could live in a `[storage]` section or at the top level. Default: `"_"`. Not written to `mdvs.toml` by default (hidden default, like `provider = "model2vec"`). User can add it explicitly if they have frontmatter fields that collide with `_file_id`, `_filename`, etc.

2. **Rename parquet columns** — apply prefix to all internal columns:
   - `files.parquet`: `_file_id`, `_filename`, `_content_hash`, `_built_at` (+ `data` stays as-is, gets removed when fields are promoted in TODO-0028)
   - `chunks.parquet`: `_chunk_id`, `_file_id`, `_chunk_index`, `_start_line`, `_end_line`, `_embedding`

3. **Store prefix in parquet metadata** — add `mdvs.internal_prefix` to build metadata. Changing the prefix requires `--force` rebuild (same semantics as changing model or chunk size).

4. **Validate reserved names** — during `init`/`update`, if inference finds a frontmatter field that collides with a prefixed internal column name (e.g. field named `_file_id` when prefix is `_`), hard-error with a clear message suggesting the user set a different `internal_prefix`.

### Files to update

- `src/index/storage.rs` — column names in read/write, metadata keys
- `src/index/backend.rs` — column references in SQL queries
- `src/search.rs` — column references in SQL queries
- `src/schema/config.rs` — add `internal_prefix` field (with serde default)
- `src/cmd/init.rs`, `src/cmd/update.rs` — reserved name validation
- `src/cmd/build.rs` — prefix change detection via metadata
