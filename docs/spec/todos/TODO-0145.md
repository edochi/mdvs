# TODO-0145: Support regex pattern constraints on string fields

## Original Scope

Add a `pattern` constraint to String and Array(String) fields that validates values against a regex.

## Resolution

Subsumed by [TODO-0149](TODO-0149.md). Pattern constraints will be implemented as the JSON Schema `pattern` keyword with validation delegated to `jsonschema` (which uses `fancy-regex` for ECMA-262 regex). No hand-rolled validation code or direct `regex` crate dependency needed.
