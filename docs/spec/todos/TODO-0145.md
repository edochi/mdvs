# TODO-0145: Support regex pattern constraints on string fields

## Summary

Add a `pattern` constraint to String and Array(String) fields that validates values against a regex. Builds on the constraint infrastructure scaffolded in TODO-0006.

## Details

### Constraint kind

New `ConstraintKind::Pattern` variant:

```rust
enum ConstraintKind {
    Categories(Vec<toml::Value>),
    Range { min: Option<toml::Value>, max: Option<toml::Value> },
    Length { min_length: Option<usize>, max_length: Option<usize> },
    Pattern(String),  // ← this TODO
}
```

`validate_for_type()`: returns `None` for String and Array(String), `Some(ConfigError)` for all other types explaining that pattern constraints only apply to string fields.

`validate_value()`: checks value matches the regex via `Regex::is_match()`. For Array(String), checks each element. Returns `Some(ViolationKind::PatternMismatch)` on failure.

`conflicts_with()`: returns `Some(ConfigError)` when paired with `Categories` on the same field (redundant — enumerated values already constrain the value space). Returns `None` when paired with `Length` (orthogonal — pattern constrains content, length constrains size). Returns `None` when paired with `Range` (different type domains, no overlap).

### Serde layer

New field on `Constraints` struct:

```rust
struct Constraints {
    // ...existing fields...
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pattern: Option<String>,
}
```

`active()` produces `ConstraintKind::Pattern` when `pattern` is `Some`.

### TOML representation

```toml
[[fields.field]]
name = "slug"
type = "String"

[fields.field.constraints]
pattern = "^[a-z0-9-]+$"

[[fields.field]]
name = "tags"
type = { array = "String" }

[fields.field.constraints]
pattern = "^[a-z]+$"  # each element must match
```

### Config-time validation

The regex itself is validated at config load time. Invalid regex syntax → `ConfigError` from `validate_for_type()`. Use `regex::Regex::new()` to check.

### Validation

New `ViolationKind::PatternMismatch`:
- **rule**: `pattern = "^[a-z0-9-]+$"`
- **detail**: `got "My Title!"`

For arrays, detail shows the offending element(s).

### Inference

Not inferred automatically — user-defined only.

### Storage/Query

No change. Constraints are validation-only.

### Dependencies

Requires `regex` crate (already a transitive dependency via several crates, but would need a direct dependency).

### Files

- `src/schema/constraints.rs` — add `pattern` to `Constraints`, add `Pattern` variant to `ConstraintKind`, implement `validate_for_type`/`validate_value`/`conflicts_with`
- `src/cmd/check.rs` — new `PatternMismatch` violation kind
