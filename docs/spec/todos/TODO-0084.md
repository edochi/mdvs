# TODO-0084: Rework update command pipeline

## Summary

Rework the update command to use the pipeline model. Update flattens build steps into its own pipeline when auto_build is enabled.

## Details

### Pipeline

```
read_config → scan → infer → write_config [→ validate → classify → load_model → embed → write_index]
```

Steps after `write_config` only run when auto_build is enabled (config or `--build` flag).

### Structs

```rust
struct UpdateProcessOutput {
    read_config: ProcessingStepResult<ReadConfigOutput>,
    scan: ProcessingStepResult<ScanOutput>,
    infer: ProcessingStepResult<InferOutput>,
    write_config: ProcessingStepResult<WriteConfigOutput>,
    // Build steps (Skipped when auto_build disabled)
    validate: ProcessingStepResult<ValidateOutput>,
    classify: ProcessingStepResult<ClassifyOutput>,
    load_model: ProcessingStepResult<LoadModelOutput>,
    embed: ProcessingStepResult<EmbedFilesOutput>,
    write_index: ProcessingStepResult<WriteIndexOutput>,
}

struct UpdateCommandOutput {
    process: UpdateProcessOutput,
    result: Option<UpdateResult>,
}
```

### Pipeline control

- `--dry-run` → write_config and all build steps are `Skipped`
- `auto_build` disabled → build steps are `Skipped`
- `--reinfer` / `--reinfer-all` affect infer step behavior, not pipeline shape
