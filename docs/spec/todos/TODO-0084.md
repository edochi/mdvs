# TODO-0084: Rework update command pipeline

## Summary

Rework the update command to use the pipeline model.

## Details

### Pipeline

```
read_config → scan → infer → write_config → [build pipeline]
```

The build sub-pipeline is optional (controlled by `auto_build` config / `--build` flag).

### Structs

```rust
struct UpdateProcessOutput {
    read_config: ProcessingStepResult<ReadConfigOutput>,
    scan: ProcessingStepResult<ScanOutput>,
    infer: ProcessingStepResult<InferOutput>,
    write_config: ProcessingStepResult<WriteConfigOutput>,
    build: ProcessingStepResult<BuildCommandOutput>,
}

struct UpdateCommandOutput {
    process: UpdateProcessOutput,
    result: Option<UpdateResult>,
}
```

### Notes

- Same nested build question as init (TODO-0083)
- `--dry-run` means write_config and build are `Skipped`
- `--reinfer` / `--reinfer-all` affect the infer step's behavior but not the pipeline shape
