# TODO-0101: Add markdown output format, rename text to pretty

## Summary

Add a `markdown` output format alongside the existing formats, and rename the current `text` format to `pretty`. The `--output` / `-o` flag becomes `pretty|markdown|json`, with `pretty` as default.

## Formats

| Format | Tables | Width | Multiline cells | Use case |
|---|---|---|---|---|
| `pretty` | box-drawing | terminal width | yes | interactive terminal use |
| `markdown` | pipe-separated | adapts to content | no, single-line | piping, docs, copy-paste |
| `json` | — | — | — | programmatic consumption |

## Markdown format details

- Standard markdown tables with `|` columns and `---` header separators
- Column widths adapt to content (no terminal width constraint)
- All cell values are single-line text
- Section headers using `##` (e.g., `## Violations`, `## New Fields`) for clear structure
- `-v` applies to markdown too — same content as pretty verbose, but rendered as markdown tables with section headers. Details TBD.

## Renaming

- `text` → `pretty` in `OutputFormat` enum
- `format_text()` → `format_pretty()` in `CommandOutput` trait
- Add `format_markdown()` to `CommandOutput` trait
- Update CLI help text, book pages, and any references to `--output text`
