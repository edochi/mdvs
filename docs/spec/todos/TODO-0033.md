# TODO-0033: Improve check command human-readable output

## Summary

The `check` command's human output is hard to read — violations are printed as long single lines with comma-separated file paths. Should use indented lists grouped by directory.

## Details

### Current output

```
result: required in ["questions/**", "books/**"]
  missing: books/High Performance.md, books/Intro to Neuro.md, books/Learning Tracker.md
```

### Desired output

Violations as indented bullet lists, grouped by directory:

```
result: required in ["questions/**", "books/**"]
  missing:
    - books/High Performance Browser Networking/High Performance Browser Networking.md
    - books/Introduction to Neuromorphic Computing/Introduction to Neuromorphic Computing.md
    - books/Introduction to Neuromorphic Computing/Learning Tracker.md
```

Or even better, a tree-like grouping:

```
result: required in ["questions/**", "books/**"]
  missing:
    books/
      High Performance Browser Networking/
        High Performance Browser Networking.md
      Introduction to Neuromorphic Computing/
        Introduction to Neuromorphic Computing.md
        Learning Tracker.md
```

### Changes

- `src/output.rs` or `src/cmd/check.rs` — update `format_human()` for check result structs
- The JSON output stays flat (structured data for scripting)
- Human output should be optimized for readability at a glance
