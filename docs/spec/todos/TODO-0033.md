# TODO-0033: Unified output format across all commands

## Summary

Overhaul CLI output: rename `human` â†’ `text`, repurpose `-v` as output detail flag (not tracing), add `--logs={info,debug,trace}` for tracing, render output with `tabled` crate, and standardize formatting across all 7 commands.

## Problems with current output

1. **Format name**: `--output human` is a bad name. Should be `text`.
2. **`-v` conflation**: `-v`/`-vv`/`-vvv` currently controls tracing (stderr debug logs), but users expect it to mean "more detail in the output".
3. **Inconsistent formatting**: init has aligned tables, check has comma-separated files, build crams stats into one parenthetical line, info explodes with 500-char glob lines.
4. **No detail mode**: no way to get config context or timing in command output.
5. **Tracing levels broken**: debug and trace produce identical output, no elapsed times at those levels.
6. **No table rendering**: all output is hand-rolled `format!()` with manual string concatenation.

## Design

See `docs/spec/output-format.md` for full rendered examples of every command.

### CLI flags

- `--output`/`-o`: `text` (default) | `json`
- `-v`: enriches output (both text and json) with expanded details + metadata footer
- `--logs={info,debug,trace}`: stderr tracing (absent = no logs)

### Output structure

**Compact** (default) = one-liner summary + table (no internal horizontal lines)
**Verbose** (`-v`) = one-liner summary + record tables (multi-column header row + spanning detail row) + footer one-liner

Verbose is an expansion of compact: same rows, plus indented detail content under each row and a metadata footer.

All user-defined values (field names, file paths, glob patterns) are quoted with `"`.

### Table rendering

- `tabled` crate with `Style::rounded()` for box-drawing borders
- `terminal_size` crate for automatic width detection â€” tables stretch to fill terminal
- Compact tables: `remove_horizontals()` â€” no internal row separators
- Verbose record tables: `ColumnSpan` for detail row spanning all columns + `BorderCorrection` for clean borders (no column dividers bleeding into spanning rows)
- Content wrapping/truncation via `Width::wrap()` + `Width::increase()`
- Two style helpers: `style_compact()` and `style_record()`

### Footer one-liner

Verbose adds a metadata footer after the tables:
```
498 files | glob: "**" | 5.2s
```
Content varies per command (files, glob, model, limit, elapsed).

### Info status fields

`config: match` / `config: changed â€” rebuild recommended` â€” compares embedding_model + chunking in mdvs.toml against parquet metadata.
`files: 498/498` â€” indexed count vs on-disk count. Mismatch indicates stale index.

### Search verbose â€” chunk text

Verbose search shows the best matching chunk per file, read from disk using `start_line`/`end_line`. Known limitation: only the top chunk per file is shown (limit applies to files, not chunks).

### Result struct pattern

Each command's result struct has two tiers:

```rust
struct CheckResult {
    // Always present (compact)
    files_checked: usize,
    violations: Vec<FieldViolation>,
    new_fields: Vec<NewField>,

    // Only with -v
    #[serde(skip_serializing_if = "Option::is_none")]
    elapsed_ms: Option<u64>,
}
```

In JSON mode, `-v` adds the same extra fields to the JSON object.

- Add `elapsed_ms` to all result structs for footer
