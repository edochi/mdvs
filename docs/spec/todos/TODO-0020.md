# TODO-0020: Add required Cargo.toml metadata for crates.io

## Summary

`cargo publish --dry-run` warns about missing metadata fields. Add all required and recommended fields so crates.io renders the package correctly.

## Details

Add to `[package]` in Cargo.toml:

- `description` — one-line summary
- `license = "MIT"` — matches the existing LICENSE file
- `repository` — GitHub URL
- `homepage` — same as repository for now
- `readme = "README.md"` — explicit for crates.io rendering
- `keywords` — up to 5: e.g. `["markdown", "search", "embeddings", "semantic", "vector"]`
- `categories` — from the [crates.io category list](https://crates.io/category_slugs), e.g. `["command-line-utilities", "text-processing"]`
