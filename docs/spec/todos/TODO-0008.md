# TODO-0008: Support value boundary constraints on numeric fields

## Summary

Add `min` and `max` constraints to Integer and Float fields for range validation during check.

## Details

**Toml representation:**
```toml
[[fields.field]]
name = "temperature"
type = "Float"
min = 273.15

[[fields.field]]
name = "rating"
type = "Integer"
min = 1
max = 5
```

**Validation (check):** If `min`/`max` are set, check that the numeric value falls within bounds. New ViolationKind (e.g. `OutOfRange`) or extend WrongType with detail like "got 0, expected >= 1".

**Inference:** Not inferred automatically — user-defined only.

**Storage/Query:** No change. Constraints are validation-only.

**Files:** `src/schema/config.rs` (add `min`/`max` to TomlField), `src/cmd/check.rs` (validate bounds)
