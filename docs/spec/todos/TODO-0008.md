# TODO-0008: Support value boundary constraints on numeric fields

## Summary

Add `min` and `max` constraints to Integer and Float fields for range validation during check. Builds on the constraint infrastructure scaffolded in TODO-0006.

## Details

### Constraint kind

New `ConstraintKind::Range` variant:

```rust
enum ConstraintKind {
    Categories(Vec<toml::Value>),
    Range { min: Option<toml::Value>, max: Option<toml::Value> },  // ← this TODO
    // ...
}
```

`validate_for_type()`: returns `None` for Integer and Float, `Some(ConfigError)` for all other types explaining that range constraints only apply to numeric fields.

`validate_value()`: checks numeric value falls within `[min, max]` bounds (inclusive). One or both bounds may be present. Returns `Some(ViolationKind::OutOfRange)` on failure.

`conflicts_with()`: returns `Some(ConfigError)` when paired with `Categories` on the same field (redundant — if you enumerate all allowed values, a range is meaningless). Returns `None` when paired with `Length` (different domains — `Range` constrains value, `Length` constrains size).

### Serde layer

New fields on `Constraints` struct:

```rust
struct Constraints {
    categories: Option<Vec<toml::Value>>,
    #[serde(default, skip_serializing_if = "Option::is_none")]
    min: Option<toml::Value>,
    #[serde(default, skip_serializing_if = "Option::is_none")]
    max: Option<toml::Value>,
    // ...
}
```

`active()` produces `ConstraintKind::Range` when either `min` or `max` is `Some`.

### TOML representation

```toml
[[fields.field]]
name = "temperature"
type = "Float"

[fields.field.constraints]
min = 273.15

[[fields.field]]
name = "rating"
type = "Integer"

[fields.field.constraints]
min = 1
max = 5
```

### Validation

New `ViolationKind::OutOfRange`:
- **rule**: `min = 1, max = 5`
- **detail**: `got 0`

### Inference

Not inferred automatically — user-defined only.

### Storage/Query

No change. Constraints are validation-only.

### Files

- `src/schema/constraints.rs` — add `min`/`max` to `Constraints`, add `Range` variant to `ConstraintKind`, implement `validate_for_type`/`validate_value`/`conflicts_with`
- `src/cmd/check.rs` — new `OutOfRange` violation kind
