# TODO-0014: Build detects manual config changes via parquet metadata

## Summary

Build should compare mdvs.toml config against parquet metadata to detect manual edits (e.g. user changed model name directly in toml). Require `--force` to confirm, same as `--set-*` flags.

## Details

- After filling missing config sections, if chunks.parquet exists, read `BuildMetadata`
- Compare model name, revision, chunk_size against toml values
- If any differ and `--force` is not set: error with message listing what changed
- If `--force` is set: proceed with full rebuild
- First build (no parquets) never needs `--force` — no metadata to compare against
- This closes the gap where `--set-*` flags require `--force` but manual toml edits don't
