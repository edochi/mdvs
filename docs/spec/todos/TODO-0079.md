# TODO-0079: Core pipeline abstractions

## Summary

Define the foundational types for the step-based pipeline model in a new `src/pipeline/mod.rs` module.

## Details

### Types

```rust
/// Trait for step output types — provides uniform text rendering.
pub trait StepOutput: Serialize {
    /// One-liner description of the step's result, e.g. "Scanned 5 files".
    fn format_line(&self) -> String;
}

/// A record of a single pipeline step's execution.
#[derive(Debug, Serialize)]
pub struct ProcessingStep<T: StepOutput> {
    /// Wall-clock time for this step in milliseconds.
    pub elapsed_ms: u64,
    /// Step-specific typed output.
    pub output: T,
}

/// The three states a step can be in.
#[derive(Debug, Serialize)]
pub enum ProcessingStepResult<T: StepOutput> {
    /// The step ran and produced output.
    Completed(ProcessingStep<T>),
    /// The step ran but hit an actual error.
    Failed(ProcessingStepError),
    /// The step didn't run because a previous step's outcome made it unnecessary.
    Skipped,
}

/// An error that occurred during a processing step.
#[derive(Debug, Serialize)]
pub struct ProcessingStepError {
    /// Whether this is a user error (bad input) or application error (internal failure).
    pub kind: ErrorKind,
    /// Human-readable error message.
    pub message: String,
}

/// Error categorization (HTTP analogy).
#[derive(Debug, Serialize)]
pub enum ErrorKind {
    /// Bad input: config not found, model mismatch, invalid flags (4xx).
    User,
    /// Unexpected internal failure: I/O errors, parquet corruption (5xx).
    Application,
}
```

### Text rendering

`ProcessingStepResult` provides a `format_line()` method that handles all three variants:
- `Completed` → delegates to `T::format_line()`, appends elapsed_ms if verbose
- `Failed` → renders error message
- `Skipped` → renders "skipped"

This is the single place where step rendering logic lives. Commands don't repeat it.

### No context field

Step-specific metadata (glob pattern, model name) lives inside `T`, not in the wrapper. `elapsed_ms` is the only universal field.
