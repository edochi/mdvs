# TODO-0091: Consistent output rules for text and JSON formats

## Summary

Define and enforce consistent output rules across all commands for both text and JSON formats, based on verbose flag and error state. Currently the process section only appears in JSON output and there's no uniform contract for what's shown when.

## Details

### Output rules

#### JSON format

- **No errors, compact (`--output json`)**: show only the result, wrapped in a `"result"` key
  ```json
  {"result": {"files_scanned": 43, "added": [], ...}}
  ```
- **No errors, verbose (`--output json -v`)**: show both process and result
  ```json
  {"process": {...}, "result": {...}}
  ```
- **Errors (always, regardless of verbose)**: show full output with both process and result (result may be null)
  ```json
  {"process": {"read_config": {"status": "failed", ...}, ...}, "result": null}
  ```

#### Text format

- **No errors, compact (`--output text`)**: show only result-derived content (summary line, tables)
- **No errors, verbose (`--output text -v`)**: show process step lines (with elapsed times) followed by result content
- **Errors (always, regardless of verbose)**: show process info highlighting where and why the pipeline stopped

### Implementation

1. Add `format_json(&self, verbose: bool) -> String` to `CommandOutput` trait
2. Update `print()` default to dispatch JSON to `format_json` instead of raw `serde_json::to_string_pretty`
3. Each `*CommandOutput` implements `format_json`:
   - If `result.is_some() && !verbose` → serialize `{"result": ...}` only
   - Otherwise → serialize full struct (process + result)
4. Update `format_text` on each `*CommandOutput`:
   - If `result.is_some() && verbose` → render step lines then result
   - If `result.is_some() && !verbose` → render result only (current behavior)
   - If `result.is_none()` → render process failure info (current behavior)

### Affected commands

All 7: init, build, check, update, search, info, clean.
