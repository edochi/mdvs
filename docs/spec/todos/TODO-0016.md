# TODO-0016: LanceDB backend via compile-time feature flag

## Summary

Add a LanceDB+Lance storage backend as an alternative to the current DataFusion+Parquet backend. Activated via `--features lancedb` compile-time flag. Uses ANN indexing (IVF-PQ/IVF-HNSW) instead of brute-force cosine similarity.

## Details

The `IndexBackend` trait is already in place (`src/index/backend.rs`). Currently only `ParquetBackend` implements it. This TODO adds:

1. **`LanceBackend`** implementing `IndexBackend` — stores data in `.lance` format, uses LanceDB's embedded vector search
2. **Feature flag** in `Cargo.toml`: `lancedb = ["dep:lancedb"]` with `lancedb` as optional dependency
3. **Backend dispatch** — commands currently create `ParquetBackend::new(path)` internally. With LanceDB, either:
   - Make commands generic over `impl IndexBackend` and dispatch from main.rs
   - Use a type alias with `#[cfg(feature = "lancedb")]`
4. **Config** — may need a way to indicate which backend was used to build (in metadata), so we don't accidentally read Lance data with Parquet backend or vice versa

### Key considerations

- LanceDB is embedded (runs in-process), uses DataFusion internally, returns Arrow RecordBatch
- `search` method already handles SQL WHERE clause — LanceDB supports SQL filtering alongside vector search
- Lance format is append-friendly, which could benefit incremental build (TODO-0011)
- ANN indexing trades recall for speed — only matters at scale (thousands+ of files)
- Default install (`cargo install mdvs`) should NOT require LanceDB — Parquet backend is the default
