# TODO-0002: Add .mdvsignore and .gitignore support

## Summary

Replace `walkdir` with the `ignore` crate in `scan.rs` to support `.mdvsignore` files and `.gitignore` integration. Add `skip_gitignore` setting to `[scan]` config.

## Details

**Problem:** The scan glob `**` picks up files that aren't vault content (e.g. `.claude/`, `.gemini/`, `.trash/`). Users need a way to exclude paths.

**Design:**
- Always read `.mdvsignore` (gitignore syntax, all directory levels)
- Read `.gitignore` by default (all directory levels, like git does)
- `skip_gitignore = true` in `[scan]` disables `.gitignore` reading
- `--skip-gitignore` CLI flag on `init` persists this setting

**Implementation:**
- Replace `walkdir::WalkDir` with `ignore::WalkBuilder` in `scan.rs`
- Add `skip_gitignore: bool` to `ScanConfig` in `config.rs` (default `false`)
- Add `--skip-gitignore` flag to `Init` in `main.rs`
- Pass `skip_gitignore` through to `ScannedFiles::scan()`
- May be able to drop `walkdir` dependency entirely

**Files:** `src/discover/scan.rs`, `src/schema/shared.rs` (ScanConfig), `src/main.rs`, `Cargo.toml`
