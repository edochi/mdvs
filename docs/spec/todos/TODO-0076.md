# TODO-0076: Ergonomic --where queries for field names with spaces

## Summary

Field names with spaces (common in Obsidian vaults) require double-quoted SQL identifiers in `--where` clauses, which clashes with shell quoting and creates a bad UX: `--where "\"my field\" = 'value'"`.

## Details

Options to explore:
1. **Auto-alias in VIEW**: map `my field` → `my_field` (risk: collides with existing fields)
2. **User-defined aliases in mdvs.toml**: `alias = "my_field"` on `[[fields.field]]`
3. **Accept and document**: users write quoted identifiers

Also consider how spaces interact with init/update warnings (TODO-0072) — fields with spaces should be flagged alongside fields with quotes.

## Files

- `src/search.rs` — VIEW creation
- `src/schema/config.rs` — potential alias field
