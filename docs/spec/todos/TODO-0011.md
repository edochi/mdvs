# TODO-0011: Incremental build

## Summary

Build currently does a full re-chunk + re-embed on every run. Implement incremental builds that only re-embed changed files using content hashes.

## Details

- On build, compare content hashes from `files.parquet` against current files on disk
- Only re-chunk and re-embed files whose content hash changed (or are new)
- Remove rows for files that no longer exist
- Schema-only changes (fields added/removed) should rewrite `files.parquet` without re-embedding
- Model/config changes still require full rebuild (already gated by `--force`)
- Content hashes are already computed (`content_hash` in `FileRow`) — need to read existing parquet and diff
