# TODO-0108: --set-revision with empty string or "None" should clear the revision

## Summary

`mdvs build --set-revision "" --force` writes `revision = ""` to `mdvs.toml` instead of removing the field. Both `""` and `"None"` should be treated as "unset" and remove the `revision` key entirely (relying on `#[serde(skip_serializing_if = "Option::is_none")]`).

## Details

Currently `--set-revision` takes a `String` and stores it as `Some(value)`. When the value is empty or literally `"None"`, it should map to `None` so serde skips serialization.

- `src/cmd/build.rs` — `mutate_config()` where `--set-revision` is applied
