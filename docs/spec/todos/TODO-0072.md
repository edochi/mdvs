# TODO-0072: Escape special characters in search SQL construction

## Summary

Special characters in field names and values can break the SQL strings constructed for DataFusion queries. Field names with quotes break the `CREATE VIEW` SQL in `search.rs`. Values with quotes (e.g. `O'Brien`) break user-supplied `--where` clauses.

## Details

Two sub-problems:

1. **Field names in VIEW SQL** (`search.rs:171-174`): Field names are interpolated directly into the `CREATE VIEW` statement — both in the dictionary accessor (`_data['...']`) and the column alias (`AS "..."`). A field named `author's_note` produces invalid SQL and crashes search. Fix: escape single quotes in the accessor (double them: `''`) and double quotes in the alias (double them: `""`).

2. **Values in `--where` clauses**: Users writing `--where "author = 'O'Brien'"` get broken SQL. This is inherent to raw SQL — the user must know SQL escaping conventions. Options: document the escaping rules, or provide a thin wrapper that handles common cases. Note: DataFusion parameterized queries could help if supported.

Consider also rejecting field names with control characters (newlines, tabs, null bytes) during inference.
