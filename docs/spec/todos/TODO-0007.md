# TODO-0007: Support Date field type

## Summary

Add a Date type to FieldType for date-valued frontmatter fields. Currently dates are treated as plain strings with no special inference, validation, storage, or query support.

## Details

**Inference:** Heuristic detection — if all non-null values of a string field match a date pattern (ISO 8601, YYYY-MM-DD, etc.), infer as Date. Need to define which formats are recognized. Note: Obsidian wiki-link dates like `"[[2026-02-22]]"` would need preprocessing (see TODO-0009) before date inference can work on them.

**Validation (check):** Verify that values parse as valid dates in the expected format. Could support a `date_format` constraint on the field.

**Storage:** Arrow `Date32` or `Timestamp(Microsecond, None)` column instead of Utf8. Enables efficient date comparisons in DataFusion.

**Query:** Date-aware WHERE clauses — comparisons like `data['date'] > '2026-01-01'` would work natively with Arrow date types instead of string comparison.

**FieldType changes:** Add `FieldType::Date` variant. Impacts: `field_type.rs`, `shared.rs` (FieldTypeSerde), `storage.rs` (Arrow schema), `check.rs` (type_matches), `infer.rs` (detection heuristic).

**Open questions:**
- Which date formats to support? Just ISO 8601, or configurable?
- Should Date include time (datetime) or be date-only?
- How to handle timezone-naive vs timezone-aware dates?
