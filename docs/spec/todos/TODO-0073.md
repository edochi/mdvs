# TODO-0073: Build violation output goes to stderr instead of stdout

## Summary

When `build` aborts due to validation violations, the formatted check result (tables) is embedded in `anyhow::bail!()`, sending it to stderr as unstructured error text. For `--output json`, no JSON is produced at all.

## Details

`check` handles this correctly: prints the result to stdout via `CommandOutput::print()`, then calls `std::process::exit(1)`. `build` should follow the same pattern — print the `CheckResult` to stdout first, then bail. This ensures:
- Formatted tables go to stdout (not stderr)
- `--output json` produces a valid JSON result
- Consistent behavior between `check` and `build`

## Files

- `src/cmd/build.rs` — change violation handling (lines 448-451)
- `src/main.rs` — may need to handle build exit code like check
