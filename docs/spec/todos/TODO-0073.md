# TODO-0073: Build violation output goes to stderr instead of stdout

## Summary

When `build` aborts due to validation violations, the formatted check result (tables) is embedded in `anyhow::bail!()`, sending it to stderr as unstructured error text. For `--output json`, no JSON is produced at all.

## Resolution

Added `BuildOutcome` enum to `build.rs` with `Success(BuildResult)` and `ValidationFailed(CheckResult)` variants. `build::run()` returns `Ok(BuildOutcome::ValidationFailed(check_result))` instead of bailing. `main.rs` matches on the outcome — prints `CheckResult` to stdout via `CommandOutput::print()`, then calls `process::exit(1)`. Same pattern as `check`.

`init.rs` and `update.rs` also updated to handle `BuildOutcome` — validation failure after inference is treated as a bug (bails with descriptive message).

Removed `depends_on: [78]` — this fix is independent. TODO-0078 tracks the broader step-based output model.
