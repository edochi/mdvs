# TODO-0028: Bare field names in --where clauses

## Summary

Let users write `--where "draft = false"` instead of `--where "data['draft'] = false"`. Frontmatter fields should be queryable as top-level column names.

## Resolution

Implemented via a DataFusion view (`files_v`) created in `SearchContext::new()`. After registering the `files` parquet table, the code inspects the `_data` Struct column's children from the Arrow schema and generates:

```sql
CREATE VIEW files_v AS
SELECT *, _data['title'] AS title, _data['draft'] AS draft, ...
FROM files
```

The search SQL joins against `files_v` instead of `files`. Bare field names resolve naturally. The old `_data['field']` bracket syntax still works via `SELECT *` for backward compatibility.

Updated help text (`main.rs`), README, spec, and tests to use bare field names.
