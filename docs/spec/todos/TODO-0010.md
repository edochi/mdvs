# TODO-0010: Support length constraints on strings and arrays

## Summary

Add `min_length` and `max_length` constraints to String and Array fields for length validation during check. Builds on the constraint infrastructure scaffolded in TODO-0006.

## Details

### Constraint kind

New `ConstraintKind::Length` variant:

```rust
enum ConstraintKind {
    Categories(Vec<toml::Value>),
    Range { min: Option<toml::Value>, max: Option<toml::Value> },
    Length { min_length: Option<usize>, max_length: Option<usize> },  // ← this TODO
}
```

`validate_for_type()`: returns `None` for String and Array, `Some(ConfigError)` for all other types explaining that length constraints only apply to String and Array fields.

`validate_value()`: checks string `.chars().count()` or array `.len()` falls within `[min_length, max_length]` bounds (inclusive). One or both bounds may be present. Returns `Some(ViolationKind::LengthOutOfBounds)` on failure.

`conflicts_with()`: returns `None` when paired with `Categories` on Array fields (orthogonal — `Categories` constrains element values, `Length` constrains array size). Returns `None` when paired with `Range` (different type domains). Returns `Some(ConfigError)` when paired with `Categories` on String fields (redundant — enumerated values already imply length bounds). Start strict, relax if needed.

### Serde layer

New fields on `Constraints` struct:

```rust
struct Constraints {
    categories: Option<Vec<toml::Value>>,
    min: Option<toml::Value>,
    max: Option<toml::Value>,
    #[serde(default, skip_serializing_if = "Option::is_none")]
    min_length: Option<usize>,
    #[serde(default, skip_serializing_if = "Option::is_none")]
    max_length: Option<usize>,
}
```

`active()` produces `ConstraintKind::Length` when either `min_length` or `max_length` is `Some`.

### TOML representation

```toml
[[fields.field]]
name = "title"
type = "String"

[fields.field.constraints]
min_length = 1
max_length = 100

[[fields.field]]
name = "tags"
type = { array = "String" }

[fields.field.constraints]
min_length = 1    # at least one tag
```

### Validation

New `ViolationKind::LengthOutOfBounds`:
- **rule**: `min_length = 1, max_length = 100`
- **detail**: `got length 0`

### Inference

Not inferred automatically — user-defined only.

### Storage/Query

No change. Constraints are validation-only.

### Files

- `src/schema/constraints.rs` — add `min_length`/`max_length` to `Constraints`, add `Length` variant to `ConstraintKind`, implement `validate_for_type`/`validate_value`/`conflicts_with`
- `src/cmd/check.rs` — new `LengthOutOfBounds` violation kind
