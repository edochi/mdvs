# TODO-0080: Shared step output structs and run functions

## Summary

Define the output structs and run functions for each processing step in `src/pipeline/` submodules. Each module owns its run function, output struct, and `StepOutput` impl.

## Details

### Module structure

```
src/pipeline/
├── mod.rs            # core types (TODO-0079)
├── read_config.rs    # run_read_config(), ReadConfigOutput
├── scan.rs           # run_scan(), ScanOutput
├── validate.rs       # run_validate(), ValidateOutput
├── infer.rs          # run_infer(), InferOutput
├── classify.rs       # run_classify(), ClassifyOutput
├── load_model.rs     # run_load_model(), LoadModelOutput
├── embed.rs          # run_embed_files(), run_embed_query(), EmbedFilesOutput, EmbedQueryOutput
├── write_index.rs    # run_write_index(), WriteIndexOutput
├── read_index.rs     # run_read_index(), ReadIndexOutput
├── write_config.rs   # run_write_config(), WriteConfigOutput
└── delete_index.rs   # run_delete_index(), DeleteIndexOutput
```

### Step output structs

Each struct implements `Serialize` and `StepOutput`.

| Module | Struct | Key fields | `format_line()` example |
|--------|--------|------------|------------------------|
| `read_config` | `ReadConfigOutput` | config path | `"Read config mdvs.toml"` |
| `scan` | `ScanOutput` | files found, glob | `"Scanned 5 files"` |
| `validate` | `ValidateOutput` | files checked, violations, new fields | `"Validated 5 files — no violations"` |
| `infer` | `InferOutput` | fields inferred, fields changed | `"Inferred 12 fields"` |
| `classify` | `ClassifyOutput` | new/edited/unchanged/removed counts | `"Classified 5 files (2 new, 3 unchanged)"` |
| `load_model` | `LoadModelOutput` | model name, revision | `"Loaded model minishlab/potion-base-8M"` |
| `embed` | `EmbedFilesOutput` | files embedded, chunks produced | `"Embedded 2 files (8 chunks)"` |
| `embed` | `EmbedQueryOutput` | query text | `"Embedded query"` |
| `search` | `ExecuteSearchOutput` | results count | `"Found 5 results"` |
| `write_config` | `WriteConfigOutput` | config path | `"Wrote mdvs.toml"` |
| `write_index` | `WriteIndexOutput` | files written, chunks written | `"Wrote index (5 files, 12 chunks)"` |
| `read_index` | `ReadIndexOutput` | metadata, file/chunk counts | `"Read index (5 files, 12 chunks)"` |
| `delete_index` | `DeleteIndexOutput` | files removed, bytes freed | `"Deleted .mdvs/ (3 files, 1.2 MB)"` |

### Run functions

Each `run_*()` function:
- Takes the inputs it needs (config, scanned files, etc.)
- Calls into existing domain logic (`src/discover/`, `src/index/`, `src/search.rs`)
- Returns `ProcessingStepResult<T>` — catches errors explicitly, returns `Failed` on error
- Measures elapsed time internally

The existing domain modules (`discover`, `index`, `search`) stay unchanged. Pipeline modules are an orchestration layer on top.
