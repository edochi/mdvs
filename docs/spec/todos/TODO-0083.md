# TODO-0083: Rework init command pipeline

## Summary

Rework the init command to use the pipeline model. Init flattens build steps into its own pipeline when auto_build is enabled.

## Details

### Pipeline

```
scan → infer → write_config [→ validate → classify → load_model → embed → write_index]
```

Steps after `write_config` only run when auto_build is enabled. When `--suppress-auto-build`, they are `Skipped`.

### Structs

```rust
struct InitProcessOutput {
    scan: ProcessingStepResult<ScanOutput>,
    infer: ProcessingStepResult<InferOutput>,
    write_config: ProcessingStepResult<WriteConfigOutput>,
    // Build steps (Skipped when --suppress-auto-build)
    validate: ProcessingStepResult<ValidateOutput>,
    classify: ProcessingStepResult<ClassifyOutput>,
    load_model: ProcessingStepResult<LoadModelOutput>,
    embed: ProcessingStepResult<EmbedFilesOutput>,
    write_index: ProcessingStepResult<WriteIndexOutput>,
}

struct InitCommandOutput {
    process: InitProcessOutput,
    result: Option<InitResult>,
}
```

### Pipeline control

- No nesting — build steps are flattened into init's pipeline
- `--suppress-auto-build` → validate through write_index are `Skipped`
- Init calls the same `run_validate()`, `run_embed_files()`, etc. as build — shared step functions
