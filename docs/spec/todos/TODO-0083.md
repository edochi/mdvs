# TODO-0083: Rework init command pipeline

## Summary

Rework the init command to use the pipeline model.

## Details

### Pipeline

```
scan → infer → write_config → [build pipeline]
```

The build sub-pipeline is optional (controlled by `--suppress-auto-build`).

### Structs

```rust
struct InitProcessOutput {
    scan: ProcessingStepResult<ScanOutput>,
    infer: ProcessingStepResult<InferOutput>,
    write_config: ProcessingStepResult<WriteConfigOutput>,
    build: ProcessingStepResult<BuildCommandOutput>,  // nested command output
}

struct InitCommandOutput {
    process: InitProcessOutput,
    result: Option<InitResult>,
}
```

### Open questions

- Build is a nested command, not a simple step. Should it be `ProcessingStepResult<BuildCommandOutput>` (a step whose output is itself a command output)? Or should init flatten the build steps into its own pipeline?
- If `--suppress-auto-build`, the build step is `Skipped` (not failed — intentionally skipped by the user)
