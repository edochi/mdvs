# TODO-0081: Rework check command pipeline

## Summary

Rework the check command to use the pipeline model.

## Details

### Pipeline

```
read_config → scan → validate
```

### Structs

```rust
struct CheckProcessOutput {
    read_config: ProcessingStepResult<ReadConfigOutput>,
    scan: ProcessingStepResult<ScanOutput>,
    validate: ProcessingStepResult<ValidateOutput>,
}

struct CheckCommandOutput {
    process: CheckProcessOutput,
    result: Option<CheckResult>,
}
```

### Rendering

- Compact success: only `result` (current behavior — "Checked 5 files — no violations")
- Verbose success: step lines + result
- Failure: step lines up to failure, no result

### Notes

- Violations are normal output in `ValidateOutput`, not errors
- Exit code 1 when violations found (same as today)
- `CheckResult` is the existing struct, mostly unchanged
