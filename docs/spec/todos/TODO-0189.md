# TODO-0189: Collapse main.rs output dispatch into CommandResult::render()

## Summary

The same 12-line `match (&cli.output, verbose) { ... }` block is duplicated nine times across `main.rs`, once per command. Adding any new `OutputFormat` variant (markdown, NDJSON, an agent-shaped compact JSON, etc.) today requires updating all nine call sites. Pull the dispatch into a single method on `CommandResult` so adding a new format becomes a two-touch change: one new enum variant in `output.rs` + one new match arm in `CommandResult::render`.

## Details

### Current shape

Every command in `main.rs` (Init, Build, Search, Check, Update, Clean, Info, ExportJsonschema, and the dry-run path) has this block:

```rust
let verbose = cli.verbose || failed;
let output_str = match (&cli.output, verbose) {
    (OutputFormat::Text, true)  => mdvs::render::format_text(&result.render_verbose()),
    (OutputFormat::Text, false) => mdvs::render::format_text(&result.render_compact()),
    (OutputFormat::Json, true)  => serde_json::to_string_pretty(&result)?,
    (OutputFormat::Json, false) => match result.result_value() {
        Some(outcome) => serde_json::to_string_pretty(outcome)?,
        None          => serde_json::to_string_pretty(&result)?,
    },
};
print!("{output_str}");
```

Verified character-for-character identical across all nine sites by the feasibility audit. Exit-code logic (`if failed { exit(2) }`, `if violations { exit(1) }`) lives outside the block and varies per command — that stays at the call site.

### Target shape

Add a method on `CommandResult` in `step.rs`:

```rust
impl CommandResult {
    pub fn render(&self, format: &OutputFormat, verbose: bool) -> Result<String, serde_json::Error> {
        Ok(match (format, verbose) {
            (OutputFormat::Text, true)  => format_text(&self.render_verbose()),
            (OutputFormat::Text, false) => format_text(&self.render_compact()),
            (OutputFormat::Json, true)  => serde_json::to_string_pretty(self)?,
            (OutputFormat::Json, false) => self.compact_json()?,
        })
    }

    fn compact_json(&self) -> Result<String, serde_json::Error> {
        match self.result_value() {
            Some(outcome) => serde_json::to_string_pretty(outcome),
            None          => serde_json::to_string_pretty(self),
        }
    }
}
```

Each call site collapses to:

```rust
print!("{}", result.render(&cli.output, cli.verbose || failed)?);
```

### Polish suggestions from the audit

- Take `&OutputFormat` instead of owned to avoid `.clone()` at call sites.
- Extract the `Json + !verbose` inner match into a small private helper (`compact_json`) for symmetry with `render_compact` / `render_verbose`.

### Out of scope

- Wiring `format_markdown` into the CLI (separate decision; can land as a follow-up two-touch change once this refactor is in).
- Adding any new `OutputFormat` variant. This TODO is the prep step; variants ship in follow-up PRs.
- Touching exit-code logic — that stays as-is at the call site, per command.

### Module dependency note

`step.rs` will gain `use crate::output::OutputFormat;` and `use crate::render::format_text;`. Verified by the audit: no cycle (output.rs and render.rs do not import from step.rs).

### Files

- `crates/mdvs/src/step.rs` — add `render()` method + private `compact_json()` helper.
- `crates/mdvs/src/main.rs` — collapse all nine dispatch sites.

### Tests

Add a unit test in `step.rs` covering all four (format, verbose) combinations on a small `CommandResult`. Existing unit tests for `render_verbose` / `render_compact` / `Serialize` keep working unchanged. No integration tests touch this surface (verified).
