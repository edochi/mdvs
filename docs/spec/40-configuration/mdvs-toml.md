# Configuration: `.mdvs.toml`

**Status: DRAFT**

**Cross-references:** [Terminology](../01-terminology.md) | [Crate: mdvs](../10-crates/mdvs/spec.md)

---

## Overview

Search-specific settings file used only by `mdvs`. Not read by `mfv`. Generated by `mdvs init`, editable by the user.

**Location:** `.mdvs.toml` in the vault root.

---

## File Format

```toml
[model]
name = "minishlab/potion-multilingual-128M"
# revision = "a1b2c3d4e5f6"
# truncate_dim = 256

[chunking]
max_chunk_size = 1000

[storage]
store_raw_content = false

[behavior]
on_search = "auto"

[search]
default_limit = 10
snippet_length = 120
```

---

## Sections

### `[model]`

| Key | Type | Default | Description |
|---|---|---|---|
| `name` | string | `"minishlab/potion-multilingual-128M"` | HuggingFace model ID. Supports both Model2Vec and ST StaticEmbedding formats. |
| `revision` | string | (latest) | Pin to a specific Git commit SHA. If omitted, the latest revision is downloaded and its SHA recorded in `vault_meta`. |
| `truncate_dim` | integer | (none) | Matryoshka dimension truncation. Only effective with models trained with MRL (e.g., `static-retrieval-mrl-en-v1`). Must be ≤ native dimension. |

**Revision pinning:** Setting `revision` prevents silent model updates from causing revision mismatch warnings. Recommended for reproducibility. See [Workflow: Model Mismatch](../30-workflows/model-mismatch.md).

**Override:** The `--model` and `--revision` global CLI flags override these values for a single invocation.

### `[chunking]`

| Key | Type | Default | Description |
|---|---|---|---|
| `max_chunk_size` | integer | `1000` | Maximum chunk size in characters |

Character-based sizing is sufficient since Model2Vec handles variable-length input. Token-precise splitting is unnecessary.

**Override:** `--chunk-size` flag on `mdvs init`.

### `[storage]`

| Key | Type | Default | Description |
|---|---|---|---|
| `store_raw_content` | bool | `false` | If `true`, adds a `raw_content VARCHAR` column to `mdfiles` storing the original file content |

Useful for tools that need the raw markdown without reading from disk (e.g., export pipelines, MCP server).

### `[behavior]`

| Key | Type | Default | Description |
|---|---|---|---|
| `on_search` | string | `"auto"` | Staleness behavior before search |

**Staleness modes:**

| Value | Behavior |
|---|---|
| `auto` | Run incremental index sync before search. Results are always fresh. Minimal overhead for unchanged vaults (hash comparison only). |
| `strict` | Error if any files have changed since last index. Forces explicit `mdvs index` before search. |

### `[search]`

| Key | Type | Default | Description |
|---|---|---|---|
| `default_limit` | integer | `10` | Default number of results (overridden by `-n`) |
| `snippet_length` | integer | `120` | Maximum characters for result snippets |

---

## Generation

`mdvs init` generates `.mdvs.toml` with default values. The user can edit it afterward.

If `.mdvs.toml` already exists when `mdvs init` runs: error. The database and config are a pair — if one exists without the other, the state is inconsistent.
