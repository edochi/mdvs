# Configuration: Field Schema (`mfv.toml` / `mdvs.toml`)

## Overview

Field schema file shared between `mfv` and `mdvs`. Defines frontmatter field types and validation rules via `allowed` and `required` glob patterns. Generated by `mfv init` (with inferred patterns from tree inference), then edited by the user.

**File naming:**
- `mfv.toml` â€” used by standalone mfv users
- `mdvs.toml` â€” used by mdvs (also discovered by mfv as fallback)

**Config discovery** (`mfv check`): `--schema` flag â†’ `mfv.toml` â†’ `mdvs.toml` â†’ error.

## File Format

### Top-Level Sections

```toml
[directory]
glob = "**"

[fields]

[[fields.field]]
name = "title"
type = "string"
```

### `[directory]` Section

| Key | Type | Default | Description |
|---|---|---|---|
| `glob` | string | `"**"` | File glob pattern for discovery and validation |

### `[[fields.field]]` Array of Tables

Each field is defined as an entry in the `[[fields.field]]` array. The `name` key identifies the frontmatter key.

| Key | Type | Default | Description |
|---|---|---|---|
| `name` | string | (required) | Frontmatter key name |
| `type` | string | `"string"` | Field type. One of: `string`, `string[]`, `date`, `boolean`, `integer`, `float`, `enum` |
| `allowed` | string[] | `["**"]` | Glob patterns where this field may appear |
| `required` | string[] | `[]` | Glob patterns where this field must be present |
| `pattern` | string | â€” | Regex the value must match (string/date types only) |
| `values` | string[] | â€” | Allowed values (only valid when `type = "enum"`) |

## Allowed and Required Patterns

### Semantics

Both `allowed` and `required` use glob patterns relative to the vault root. They follow symmetric semantics:

| Value | Meaning |
|---|---|
| `[]` (empty) | Nowhere |
| `["**"]` | Everywhere (all files, any depth) |
| `["blog/**"]` | All files under `blog/` recursively |
| `["*"]` | Files in the root directory only (not recursive) |
| `["blog/**", "notes/**"]` | Multiple patterns OR-ed |

### Defaults

When omitted from the TOML file:
- `allowed` defaults to `["**"]` â€” a defined field is allowed everywhere
- `required` defaults to `[]` â€” fields are optional by default

This makes the common case minimal:

```toml
[[fields.field]]
name = "title"
type = "string"
# title is allowed everywhere, required nowhere
```

### Invariant: `required âŠ† allowed`

A field cannot be required at a path where it isn't allowed. If `required` is non-empty but `allowed` is empty (`[]`), schema parsing fails with a validation error.

### `*` vs `**`

- `*` matches files in a single directory (shallow)
- `**` matches files at any depth (recursive)

Generated by `mfv init` via tree inference: leaf nodes (direct files in a directory) produce `*`, directory nodes (aggregated subtrees) produce `**`.

## Type System

### Supported Types

| Type | YAML Example | Notes |
|---|---|---|
| `string` | `title: "My Post"` | Default type when omitted |
| `string[]` | `tags: [rust, crdt]` | YAML/TOML sequences |
| `date` | `date: 2025-06-12` | ISO 8601 dates |
| `boolean` | `draft: true` | YAML/TOML booleans |
| `integer` | `priority: 3` | Whole numbers |
| `float` | `score: 0.95` | Floating point |
| `enum` | `status: draft` | Validated against `values` list |

### Type Inference

When `type` is omitted, it defaults to `"string"`. During `mfv init`, types are inferred from observed values. See [mdvs-schema: Type Inference](../10-crates/mdvs-schema/spec.md#type-inference).

## Validation Rules

### `allowed` â€” Where a field may appear

```toml
[[fields.field]]
name = "doi"
type = "string"
allowed = ["papers/**"]
```

If a file outside `papers/` has a `doi` field, validation reports an error. If `allowed` is omitted (defaults to `["**"]`), the field may appear in any file.

### `required` â€” Where a field must appear

```toml
[[fields.field]]
name = "title"
type = "string"
required = ["**"]
```

Every file must have a `title` field. If `required` is omitted (defaults to `[]`), the field is optional everywhere.

Path-scoped requirement:

```toml
[[fields.field]]
name = "status"
type = "string"
required = ["blog/**"]
allowed = ["blog/**"]
```

Only files under `blog/` must have `status`. Files elsewhere are not checked.

### `pattern`

```toml
[[fields.field]]
name = "doi"
type = "string"
pattern = "^10\\.\\d{4,9}/.*"
allowed = ["papers/**"]
required = ["papers/**"]
```

Regex the string value must match. Only valid for `string` and `date` type fields. Uses Rust regex syntax.

### `values`

```toml
[[fields.field]]
name = "status"
type = "enum"
values = ["draft", "review", "published"]
required = ["blog/**"]
allowed = ["blog/**"]
```

Allowed values for enum fields. Only valid when `type = "enum"`. The value must exactly match one of the listed strings.

## Examples

### Minimal (no validation rules)

```toml
[directory]
glob = "**"

[[fields.field]]
name = "title"
type = "string"

[[fields.field]]
name = "tags"
type = "string[]"

[[fields.field]]
name = "date"
type = "date"
```

All three fields are allowed everywhere, required nowhere.

### With validation

```toml
[directory]
glob = "**"

[[fields.field]]
name = "title"
type = "string"
required = ["**"]

[[fields.field]]
name = "tags"
type = "string[]"
allowed = ["blog/**", "notes/**"]
required = ["blog/**"]

[[fields.field]]
name = "status"
type = "enum"
values = ["draft", "review", "published"]
allowed = ["blog/**"]
required = ["blog/**"]

[[fields.field]]
name = "doi"
type = "string"
pattern = "^10\\.\\d{4,9}/.*"
allowed = ["papers/**"]
required = ["papers/**"]
```

### Unknown sections ignored

`mdvs.toml` may include search-specific sections. `mfv` ignores them:

```toml
[directory]
glob = "**"

[model]
name = "minishlab/potion-base-2M"

[[fields.field]]
name = "title"
type = "string"
```

## Lock File (`mfv.lock`)

Auto-generated by `mfv init`. Captures raw per-file observations (like `Cargo.lock`). Not edited by the user.

### Format

```toml
# Auto-generated by mfv init. Do not edit.
# To regenerate: mfv init --force

[discovery]
total_files = 12
files_with_frontmatter = 11
glob = "**"
generated_at = "2025-06-12T10:00:00"

[[field]]
name = "title"
type = "string"
files = ["blog/a.md", "blog/b.md", "notes/c.md"]

[[field]]
name = "tags"
type = "string[]"
files = ["blog/a.md"]
```

### Sections

**`[discovery]`**: Metadata about the scan.

| Key | Type | Description |
|---|---|---|
| `total_files` | integer | Total markdown files matched by the glob |
| `files_with_frontmatter` | integer | Files that had parseable frontmatter |
| `glob` | string | Glob pattern used |
| `generated_at` | string | ISO 8601 timestamp |

**`[[field]]`**: One entry per discovered field.

| Key | Type | Description |
|---|---|---|
| `name` | string | Field name |
| `type` | string | Inferred type |
| `files` | string[] | Exhaustive list of files containing this field |

No derived quantities â€” count is `files.len()`.

---

## Generation Behavior

### `mfv init`

Generates `mfv.toml` with:
- `[directory]` section with the glob pattern
- `[[fields.field]]` entries with inferred types and `allowed`/`required` patterns from tree inference
- No `pattern` or `values` rules (user adds them manually)

Also generates `mfv.lock` with per-file observations.

### Existing File

If the config file already exists when `mfv init` runs, it errors with a message suggesting `--force`. With `--force`, it overwrites both the config and lock files.
